## 二、Step 0 · 输入预处理

### 三级输入自适应

| 层级 | 输入条件 | 预处理策略 |
|------|----------|------------|
| **Tier 1** | 完整材料（录音稿+精简稿+讲稿+信息图） | 直接进入 Step 0.1，无需额外处理 |
| **Tier 2** | 仅有录音稿 | 执行清洗：去口语化、语义分段、提取精简稿，再进入 Step 0.1 |
| **Tier 3** | 仅有课程标题/主题 | 需用户提供基本方向（目标受众、学时、核心要点），AI 生成初稿框架待确认 |

### Tier 2 清洗规范

1. **去口语化**：删除"嗯""那个""就是说"等填充词，保留原意
2. **语义分段**：按主题切分为结构化段落，添加小标题
3. **提取精简稿**：每段提炼3-5个核心要点，生成 Markdown 精简稿
4. **缺口标注**：清洗后材料不足的部分，标记 `[待补充]`

### 来源标注体系（全流程通用）

> v1.2 升级：5级标注 + 出处字段，按内容安全级别动态启用

#### 标注级别与内容安全级别对应

| 内容安全级别 | 启用的标注 | 出处要求 |
|-------------|-----------|---------|
| **标准** | `[原]` `[改]` `[补]` | 简略出处（文件名+段落号 / 来源名称+日期） |
| **专业** | `[原]` `[改]` `[引]` `[补]` `[待核]` | 详细出处（文件名+日期+段落号 / 网页URL） |
| **党政** | `[原]` `[改]` `[引]` `[补]` `[脱敏]` `[待核]` | 严格出处（文件名+日期+段落号 / 网页URL + 领导人名称必须脱敏） |

#### 标注定义

| 标记 | 含义 | 使用场景 | 出处要求 |
|------|------|----------|----------|
| `[原]` | 录音稿原文，未做修改 | 原始材料直接引用 | 录音稿文件名+段落号 |
| `[改]` | 基于录音稿改写 | 调整结构、精简语言、逻辑重组 | 录音稿文件名+段落号+"基于原文改写" |
| `[引]` | 引用政策文件/讲话原文 | 涉及政策表述、领导人讲话、会议文件 | **文件名+发布日期+段落号 / 或网页URL** |
| `[补]` | 补充内容（非原文非政策文件） | 教学设计层面的补充：研讨议题、时间安排 | 信息来源名称+日期 / 或网页URL |
| `[脱敏]` | 因内容安全要求省略的敏感表述 | 党政类课程中领导人名称等 | **[脱敏：原文为XX，因内容安全要求省略]** |
| `[待核]` | 暂无法确认出处 | 涉及事实判断、数据引用、政策版本存在疑问时 | **"暂无法确认出处，需人工核实"** |

#### 规则

- 涉及政策表述、领导人讲话、会议文件 → 只能用 `[引]`，不能用 `[补]`
- `[补]` 只能用于教学设计层面，不能用于事实性内容
- `[脱敏]` 仅在党政模式下出现，领导人名称使用"中央领导同志"/"相关领导同志"替代
- `[待核]` 标记在交付前必须逐条核实或删除

#### 标注样式示例

```
干部要具备七种能力
[引]《在某干部学院专题培训班开班式上的讲话》2020年10月

改革推进中要做到"五个有"
[原]录音稿_提高改革攻坚能力.md 第12段

2024年全国GDP增长5.0%
[补]国家统计局2024年国民经济统计公报 https://...

中央领导同志在某干部学院专题培训班开班式上提出干部要具备七种能力
[脱敏：原文为XX同志，因内容安全要求省略]

某些地方改革试点效果显著
[待核]暂无法确认出处，需人工核实
```

## 三、Step 0.1 · 课程参数选择（启动包）

> v1.2 变更：从"课程参数选择面板"升级为"课程工作流启动包"。新增内容安全级别（3档预设）、文件上传区、交互脚本生成。

### 使用工具

`课程工作流启动包.html`（单文件HTML，浏览器直接打开即可使用）

### 参数维度

| 维度 | 选项 | 说明 |
|------|------|------|
| **课程名称** | 用户输入 | 课程标题 |
| **课程类型** | A型（讲授式）/ B型（互动案例讨论式） | 决定产出数量和结构 |
| **讲师风格** | 专家讲授型 / 实战教练型 / 故事启发型 / 咨询顾问型 / 互动引导型 | 影响逐字稿语调、PPT信息密度、互动设计 |
| **课程时长** | 半天课（150分钟）/ 一天课（300分钟）/ 自定义（60-420分钟） | 决定时间轴结构和环节数量；午休不计入课程时长 |
| **内容安全级别** | 标准 / 专业 / 党政 | 3档预设，决定防幻觉强度、出处标注详细度、是否脱敏、是否强制审校 |
| **课程材料** | 文件上传 | 支持拖拽上传 .md/.txt/.docx/.pdf/.pptx |

### 内容安全级别（3档预设）

| 档位 | 名称 | 适用场景 | 规则组合 |
|------|------|---------|---------|
| 1档 | **标准** | 企业培训、技能课、创新课 | 基础防幻觉 + 简略出处标注（3级） |
| 2档 | **专业** | 学术课程、公共管理、专业认证 | 严格防幻觉 + 详细出处标注（5级）+ 二次审校可选 |
| 3档 | **党政** | 党政类课程、干部培训 | 严格防幻觉 + 详细出处标注（5级+脱敏）+ 脱敏处理 + 二次审校强制 |

#### 防幻觉规则

| 级别 | 规则 |
|------|------|
| **基础版**（标准） | 不得杜撰任何不存在的事实和数据；无法确认的内容标注[待核]；补充内容标注来源 |
| **严格版**（专业/党政） | 不得杜撰任何不存在的事实、数据、文件名、会议名、讲话内容；无法确认的内容必须标注[待核]，不得伪装为确定信息；所有政策表述必须标注出处；所有数据引用必须标注出处（年份+地区+口径+来源） |

#### 脱敏规则（仅党政模式）

1. 涉及党和国家领导人时，使用"中央领导同志"/"相关领导同志"替代具体姓名
2. 保留会议名称和日期，省略具体姓名
3. 涉及敏感政治术语时，用学术化、概括化表述替代
4. 脱敏处标注 `[脱敏：原文为XX，因内容安全要求省略]`
5. 脱敏不影响课程内容的语义完整性——教学目标、方法论、案例逻辑不依赖具体姓名

### 讲师风格对生成的影响

| 风格 | 逐字稿影响 | PPT影响 | 互动影响 |
|------|-----------|---------|---------|
| **专家讲授型** | 逻辑严谨，术语定义完整 | 信息密度高，论证链条保留 | 互动点少（2-4个） |
| **实战教练型** | 带练口吻，每个模块配操作任务 | 案例密集，步骤清晰 | 互动点多（4-6个） |
| **故事启发型** | 故事钩子，场景化表达 | 金句页增加，长段文字减少 | 互动点适中（3-5个） |
| **咨询顾问型** | 先给结论再展开依据 | 图表和矩阵增多 | 互动点适中（3-5个） |
| **互动引导型** | 问题链、控场提示、收束话术 | 信息密度降低，研讨页增多 | 互动点最多（5-8个） |

### 输出：课程工作流启动包

选择完成后，点击"生成课程启动包"按钮，输出结构化文本，包含：
- 课程参数（名称、类型、风格、时长、安全级别）
- 上传材料清单
- **5轮交互脚本**（AI引导用户逐轮确认参数→上传录音稿→上传参考材料→填写想法→启动生成）
- 内容安全规则（按安全级别写入对应规则）
- 来源标注规则（按安全级别写入对应标注体系）
- 产出规范
- 质量检查清单

启动包复制到AI对话窗口后，AI按交互脚本逐轮引导用户完成材料收集，然后执行全流程生成。

## 四、5轮交互引导（Step 0.1 → Step 0）

> v1.2 新增。AI收到启动包后，按交互脚本逐轮引导用户完成材料收集，不是一股脑让用户自己上传。

### 交互流程

| 轮次 | AI动作 | 用户动作 | 兜底选项 |
|------|--------|---------|---------|
| **第1轮** | 确认工单参数 | 回复"确认"或修改参数 | — |
| **第2轮** | 引导上传录音稿 | 上传文件 | "无录音稿" → Tier 3 |
| **第3轮** | 引导上传参考材料 | 上传多个文件 | "无补充材料" |
| **第4轮** | 引导填写想法 | 文字描述侧重点/特殊要求 | "无补充" |
| **第5轮** | 展示完整流程确认 | 回复"开始" | — |

### 兜底机制

- 无录音稿 → 按 Tier 3（最小输入）模式处理
- 无参考材料 → AI从录音稿中提取可用内容，缺口标注 `[待补充]`
- 无个人想法 → 按工单参数直接生成
- 未回复"开始" → AI不执行任何生成步骤

## 五、Step 0.3 · 政策文件校准

> v1.2 变更：标准模式跳过；专业模式可选执行；党政模式强制执行。

### 触发条件

| 内容安全级别 | 是否执行 | 执行范围 |
|-------------|---------|---------|
| **标准** | 跳过 | — |
| **专业** | 可选 | 扫描政策表述和数据引用，核实出处 |
| **党政** | 强制 | 扫描全部政策表述+数据引用+领导人名称，核实出处+脱敏处理 |

### 执行内容（专业/党政模式）

1. **扫描输入材料**：识别所有政策性表述（文件名、会议名、领导人讲话、提法概念、数据引用）
2. **与知识库比对**：
   - 该表述是否存在于权威文件中？
   - 是否最新版本？是否有更新提法？
   - 数据年份、口径、来源是否明确？
3. **脱敏处理（仅党政模式）**：
   - 领导人具体姓名 → "中央领导同志"/"相关领导同志"
   - 保留会议名称和日期
   - 标注 `[脱敏]`
4. **输出校准报告**：

| 状态 | 含义 | 处理方式 |
|------|------|----------|
| ✓ 已核实 | 表述准确，出处清晰 | 标注 `[引]` + 文件名+日期+段落 |
| ⚠ 提法需更新 | 存在更新版本 | 建议替换为最新提法，附出处 |
| ✗ 无法核实 | 找不到权威来源 | 标记 `[待核]`，交付前必须处理 |

### 政策核验表（交付标准）

| 核验项 | 执行规则 | 交付标注 |
|--------|----------|----------|
| **政策提法** | 优先检索近期正式文件、政府官网和权威发布，多方对比 | 标注最新提法、文件名、发布日期 |
| **数据口径** | 优先使用最新年度统计公报、白皮书或主管部门数据 | 标注年份、地区、口径和来源链接 |
| **历史资料** | 保留教学价值时必须说明历史语境，不能冒充当前表述 | 标注历史年份和适用边界 |
| **媒体解读** | 只能作为辅助理解，不替代正式政策和原始数据 | 标注为解读材料，避免结论化引用 |
| **领导人名称**（党政模式） | 使用替代表述，省略具体姓名 | 标注 `[脱敏]` |

### 知识库来源（可选）

- ima知识库（当前已连接）：可导入权威文件原文
- 手动维护：`课程/政策文件库/` 目录，按年份和主题分类

## 八、各产出详细规范

### Step 1 · 课程大纲

**文件名**：`01_课程大纲.md`

**结构要求**：
- 课程信息头（类型、学时、教学方式、讲师风格、内容安全级别、来源标记说明）
- 教学目标表（认知/分析/实践三级）
- 章节时间轴（每环节标明时长、内容、对应页码）
- A型：理论章节结构
- B型额外含：议题清单、角色卡、评估表、讨论计时安排
- 页面清单（演示文稿每页内容+版式类型）
- 材料来源说明

**关键检查点**：
- B型分组数量统一为 **4组**（不写5组/6组）
- 每环节时间加总 = 总学时
- 页面清单页码与逐字稿 `[PAGE X]` 一一对应
- 来源标注使用正确（按安全级别对应标注体系）
- 党政模式：领导人名称已脱敏

### Step 2 · 逐字稿

**文件名**：`02_逐字稿.md`

**结构要求**：
- `[PAGE X]` 标记对应演示文稿页码
- B型额外含：

| 5. 是否有与已知政策文件矛盾的表述？ | — | ✅ | ✅ |

| 状态 | 含义 | 处理 |
|------|------|------|
